# Android — Project Intelligence

## Architecture

This is a **Capacitor WebView shell** — a thin native wrapper that loads the React
web app (`web/src/`) from bundled assets. There is no native Kotlin/Compose UI.

```
MainActivity (Java)
  └── BridgeActivity (Capacitor)
        └── WebView → React app (web/dist/)
```

All UI, auth, data fetching, and business logic live in the web layer.

## Build order

```bash
# From web/
npm run build           # 1. compile React → web/dist/
npx cap sync android    # 2. copy assets + regenerate capacitor.settings.gradle
cd android
./gradlew :app:assembleDebug  # 3. build APK
```

## Key files

| File | Purpose |
|---|---|
| `app/src/main/java/com/ergdashboard/android/MainActivity.java` | Extends `BridgeActivity` — no Compose, no native UI |
| `variables.gradle` | SDK versions: minSdk=26, compile/targetSdk=36, AndroidX lib versions |
| `capacitor.settings.gradle` | Auto-generated by `cap sync`; includes `:capacitor-android` from node_modules |
| `app/src/main/assets/public/` | Auto-populated by `cap sync` from `web/dist/`; gitignored |

## Constraints

- Do not add Kotlin/Compose files — the WebView is the UI
- Do not add supabase-kt — auth is handled by `@supabase/supabase-js` in the web layer
- Do not commit `app/src/main/assets/public/` — generated by `cap sync` in CI
- Do not edit `capacitor.settings.gradle` — regenerated each `cap sync`

## Adding native features

Install a Capacitor plugin in `web/`:

```bash
cd web && npm install @capacitor/some-plugin
npx cap sync android
```

Then use it from React via the plugin's JS API.

## CI

`.github/workflows/ci-android.yml` runs two jobs:

1. `build-web`: `npm run build` with Supabase secrets → uploads `web/dist/` artifact
2. `build-apk`: downloads artifact → `cap sync` → `./gradlew assembleDebug` → uploads APK

Capacitor CLI version: `@capacitor/android@^8.4.1` (from `web/package.json`).
